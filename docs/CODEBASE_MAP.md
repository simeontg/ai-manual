### Environment variables (discovered)
The repo references both Vite-style `VITE_` variables for client-side bundling and server-side `process.env` variables. Files that check for these variables include `src/integrations/supabase/client.ts` and `src/integrations/supabase/client.server.ts`.

Client / public variables (safe to surface in `.env` locally):
- `VITE_SUPABASE_URL` or `SUPABASE_URL` (the project URL)
- `VITE_SUPABASE_PUBLISHABLE_KEY` or `SUPABASE_PUBLISHABLE_KEY` (anon/publishable key)
- `VITE_SUPABASE_PROJECT_ID`

Server / sensitive variables (must be kept secret):
- `SUPABASE_SERVICE_ROLE_KEY` (service role; grants DB write/bypass RLS) — DO NOT commit to repo or share.

Quick checklist before running locally
1. Copy `.env` into `.env.local` (or otherwise populate env vars) and ensure `SUPABASE_SERVICE_ROLE_KEY` is not checked into source control.
2. If you only want to run frontend without hooking a production DB, set `VITE_SUPABASE_URL` and `VITE_SUPABASE_PUBLISHABLE_KEY` to a development Supabase project or to placeholder values to avoid runtime errors in dev.
